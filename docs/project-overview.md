# プロジェクト概要 — place-finder

## プロダクトの目的

ユーザーの現在地 GPS 情報をもとに、近くの店舗をジャンルごとに検索・表示し、使いやすい形でおすすめを提示するサービス。

既存の地図アプリやグルメサイトは情報量が多すぎて「今すぐ行ける良い店」を判断しづらい。place-finder はシンプルかつ的確なおすすめ体験を提供する。

## ターゲットユーザー

- 外出先で「近くの良い店」をサッと探したい一般ユーザー
- 土地勘のない場所で飲食店や施設を探す旅行者・出張者
- 情報過多な既存サービスに疲れている層

## MVP の成立条件

「現在地から近い店をジャンルで絞り、おすすめ順で見られる」体験が一通り動くこと。

## MVP で提供する価値

- 現在地ベースで近隣店舗をジャンル別に検索・一覧表示
- ルールベースの重みづけ（距離・評価・営業中かどうか等）でおすすめ順ソート
- 店舗の基本情報（名前・ジャンル・距離・評価・営業状況）の閲覧
- 地図上での店舗位置の視覚的把握

## 初期段階でやらないこと

- ユーザー認証・アカウント管理
- お気に入り・ブックマーク機能
- レビュー投稿・ソーシャル機能
- AI ベースのパーソナライズ推薦
- プッシュ通知、多言語対応、ネイティブアプリ、管理画面

## 将来的な拡張余地

- AI 推薦への段階的移行（ルールベース → AI）
- アカウント機能（お気に入り、履歴保存）
- レビュー・口コミ機能
- 多言語対応（訪日外国人向け）
- PWA / ネイティブアプリ化

## 推薦ロジックの方針

初期段階では AI を前提とせず、ルールベース（距離・評価・営業状況の重みづけ）で十分な価値を提供する。

理由:
- AI 推薦はデータ蓄積が前提であり、初期段階ではユーザーデータが存在しない
- ルールベースでも「近い・評価が高い・営業中」の重みづけで十分有用
- ルールの調整が容易で、フィードバックを受けて素早く改善できる
- ルールベースで得たユーザー行動データが将来の AI 学習データになる
- 推薦ロジック部分を差し替え可能な設計にしておけば、AI 導入は後から可能

## 外部 API

Google Maps Platform / Places API を活用し、店舗データの取得・検索を行う。
