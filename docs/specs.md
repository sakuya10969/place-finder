# 仕様一覧 — place-finder

機能を実装するための技術仕様をここに集約する。

## 1. 現在地取得

- ブラウザの Geolocation API (`navigator.geolocation`) を使用する
- `getCurrentPosition()` で緯度・経度を取得する
- ユーザーが位置情報の許可を拒否した場合、エラーメッセージを表示し、手動での住所・地名入力にフォールバックする（将来対応候補）
- 取得した座標は API リクエストのパラメータとしてバックエンドに送信する

## 2. 店舗検索 API

- エンドポイント: `GET /api/places/search`
- バックエンドが Google Places API (New) の Nearby Search を呼び出す
- リクエストパラメータ: 緯度、経度、半径、ジャンル（オプション）
- レスポンス: 店舗リスト（名前、ジャンル、座標、評価、営業状況、距離）
- API キーはバックエンド側で管理し、フロントエンドには露出させない

## 3. ジャンル別フィルタリング

- フロントエンドでジャンル選択 UI を提供する
- 選択されたジャンルを API リクエストのパラメータとして送信する
- バックエンドで Places API の `includedTypes` パラメータにマッピングする
- 対応ジャンル（初期）: レストラン、カフェ、コンビニ、薬局、その他（拡張可能）

## 4. おすすめ順ソート（推薦ロジック）

- バックエンドの `recommendation` モジュールで実装する
- スコアリング要素:
  - 距離（近いほど高スコア）
  - 評価（高いほど高スコア）
  - 営業状況（営業中なら加点）
- 各要素に重み係数を設定し、加重スコアで降順ソートする
- 重み係数は設定値として外出しし、コード変更なしで調整可能にする

## 5. 店舗基本情報

- Places API から取得するフィールド:
  - `displayName` → 店舗名
  - `primaryType` / `types` → ジャンル
  - `location` → 座標（距離計算に使用）
  - `rating` → 評価
  - `currentOpeningHours` → 営業状況
- 距離はバックエンドで Haversine 公式等を用いて計算する

## 6. 地図表示

- Google Maps JavaScript API を使用する
- フロントエンドで直接 Maps API を呼び出す（表示専用）
- 検索結果の店舗をマーカーとして地図上に配置する
- マーカータップで店舗の基本情報をポップアップ表示する
- 現在地もマーカーで表示する

## 7. API クライアント生成

- バックエンド（Elysia + TypeBox）が OpenAPI スキーマを自動生成する
- Orval を使用して `packages/api-client/generated/` に React Query フックと型定義を生成する
- 生成コマンド: `bun run generate`
- フロントエンドは生成物のみを使用し、手書きの API 呼び出しは禁止

## 8. バリデーション

- フロントエンド: Valibot でフォーム入力・API レスポンスのランタイム検証
- バックエンド: TypeBox でリクエスト/レスポンスのスキーマ定義とバリデーション
- TypeBox スキーマが OpenAPI の Single Source of Truth

## 9. 開発環境

- パッケージマネージャー / ランタイム: Bun
- フロントエンド開発サーバー: Vite（port 5173）
- バックエンド開発サーバー: Elysia（port 3000）
- 開発時の API プロキシ: Vite の proxy 設定（`/api` → `localhost:3000`）
- リンター / フォーマッター: Biome
