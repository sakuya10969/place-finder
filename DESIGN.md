# デザイン方針 — place-finder

## コンセプト

白背景をベースに、濃い青・紺・黒の 3 色でまとめたシンプルで視認性の高い UI。
地図と店舗情報が主役であり、装飾は最小限に抑える。

## カラーパレット

| 用途 | 色 | 目安 |
|---|---|---|
| 背景 | 白 `#FFFFFF` | ページ全体、カード背景 |
| メインカラー | 濃い青 `#1A3C8A` | ボタン、アクティブ状態、リンク、主要アクション |
| サブカラー | 紺 `#0F1F4B` | ヘッダー、ナビゲーション、セクション見出し |
| テキスト / アイコン | 黒 `#1A1A1A` | 本文テキスト、アイコン、ラベル |
| ボーダー / 区切り線 | 薄いグレー `#D9D9D9` | カード枠線、セパレーター（唯一の補助色） |
| エラー | 赤 `#C53030` | バリデーションエラー、警告 |

水色、ライトブルー、パステル系は使用しない。青系は必ず濃い青 `#1A3C8A` 以上の濃度とする。

## タイポグラフィ

- 本文: system-ui, 14〜16px, 黒
- 見出し: 16〜20px, 紺または黒, font-weight 600〜700
- ラベル・補足: 12〜13px, 黒（opacity 控えめ可）
- 行間はゆとりを持たせ、情報の詰め込みすぎを避ける

## レイアウト

- Mantine の AppShell を使い、ヘッダー + メインコンテンツの構成とする
- ヘッダーは紺背景に白テキスト。ロゴとナビゲーションを配置
- メインコンテンツは白背景。左に店舗リスト、右に地図の 2 カラム構成（モバイルではタブ切り替え）
- Mantine の Grid / Flex でレスポンシブ対応する
- 余白は Mantine の spacing トークン（md, lg）を基準に統一する

## コンポーネント方針

Mantine UI のコンポーネント群をそのまま活用し、工数を削減する。

- ボタン: Mantine Button。メインカラー（濃い青）の filled バリアント。角丸は Mantine デフォルト
- カード: Mantine Card。白背景 + 薄いグレーのボーダー。shadow は使わないか最小限
- 入力: Mantine TextInput / Select。ボーダーは薄いグレー、フォーカス時に濃い青
- バッジ / チップ: Mantine Badge。ジャンルフィルタに使用。濃い青の outline バリアント、選択時は filled
- リスト: Mantine Stack + Card の繰り返し。店舗一覧の表示に使用
- モーダル: Mantine Modal。店舗詳細のポップアップ表示に使用
- ローディング: Mantine Loader / Skeleton。濃い青のカラー指定
- 通知: Mantine Notifications。エラー時は赤、成功時は濃い青

## テーマ設定

Mantine の createTheme でプロジェクト全体のテーマを定義する。

- primaryColor に濃い青を設定
- デフォルトの radius、spacing を統一
- カラースキームは light 固定（ダークモードは初期段階では対応しない）
- フォントは system-ui を指定

## 地図表示

- 地図部分は Google Maps JavaScript API で描画するため、Mantine の管轄外
- 地図コンテナは白背景のカード内に配置し、周囲のデザインと調和させる
- マーカーの色は濃い青を基調とする
- 情報ウィンドウ（InfoWindow）のスタイルは地図 API の制約内で可能な範囲で統一する

## モバイル対応

- Mantine のブレークポイント（sm, md, lg）に従ったレスポンシブ設計
- モバイルでは 2 カラムを解除し、リスト表示と地図表示をタブまたはトグルで切り替える
- タッチ操作を考慮し、ボタンやタップ領域は十分なサイズを確保する
- Mantine の Burger + Drawer でモバイルナビゲーションを実装する

## やること / やらないこと

やること:
- 白背景 + 濃い青・紺・黒の 3 色に徹底する
- Mantine コンポーネントをそのまま使い、カスタム CSS は最小限にする
- 情報の優先度に応じてメインカラー（濃い青）の使用量を制御する
- 一貫した余白・角丸・フォントサイズを維持する

やらないこと:
- 水色、ライトブルー、パステル系の色を使う
- ダークモード対応（初期段階）
- 独自のコンポーネントライブラリを構築する
- グラデーション、影の多用、過剰な装飾
